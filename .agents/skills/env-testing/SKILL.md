# 测试与验证

## 基本要求
- **用项目既有的测试框架和命令**，不擅自引入新框架。
- 改完**自测关键流程**（核心路径、刚改的边界、明显的回归点）；提交前尽量 `lint` + `build`，只跑局部要**说明哪些验证执行了、哪些没执行**。
- **优先文件级/单测命令**（单文件 lint、跑单个测试），比每次整库构建快得多。
- 「能编译」≠「功能对」：UI/交互类改动要实际运行起来看一眼。

## 环境化测试（本地 Docker 优先 + 生产数据模拟）
- **优先在本地 Docker/容器环境部署测试**：隔离、可随意重建、不碰他人和生产。用 `docker compose` / Dockerfile 一键拉起依赖（DB、缓存、消息队列），**贴近生产拓扑**、可复现可丢弃。
- **可拉取生产数据做模拟，但守红线**：
  - **脱敏** —— 清洗敏感字段（手机号、身份证、邮箱、密码哈希、token、隐私）后再灌进本地。
  - **只读隔离** —— 本地与生产物理隔离，连接串只指向本地容器，**绝不让本地代码/测试对生产库做写操作**。
  - **最小子集** —— 只拉复现所需的数据，别整库搬。
  - **不落库** —— dump / 数据文件放 `.gitignore` 内、**不提交进 git**、用完清理。
  - **合规** —— 确认有权限导出，遵守公司数据合规。
- **流程**：起依赖 → 灌脱敏数据 → build + 自动化测试 + 手动验关键流程 → 通过再推。改了 schema/迁移的，**先在本地容器验证迁移幂等与回滚**再上。
