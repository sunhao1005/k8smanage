# ai-kit 自我演进与维护

> 原则：能力靠"先找现成的"，沉淀靠"写成技能"，清理靠"提议+确认"。全程如实报告改了什么，绝不偷偷改、不无确认地删。
> **路径约定**：本文出现的 `.agents/skills/<名>/`、`.agents/hooks/` 等路径，均指**装好 kit 的目标项目**里的路径（技能沉淀到激活项目，不回写 ai-kit 母版仓的 `shared/` `code/` `chat/`）。在母版仓里读本文时注意这层区分，别误往母版目录里建技能。

## A. 缺能力时：先找现成的，别从零造
需要某能力（调某服务、跑某类分析、连某系统、解析某格式…）而手上没有时：
1. **先检索现成方案**：用 WebSearch/WebFetch 查是否有现成的 **skill / MCP 工具 / 库 / CLI**；查库用法优先用 context7 等文档源。现成技能优先检索三源：**anthropics/skills、openai/skills、awesome-claude-code**（SKILL.md 已是跨工具开放标准，可零改写复用）。
2. **核实可用再用**：看文档、版本、维护状态，确认它真存在且适配——**不编造其存在或用法**（拿不准就说不确定）。
3. **MCP 工具不自行接入**：MCP 服务器要配置命令/凭据、常需重启、涉及安全信任——**向用户说明并由用户确认接入**，不要 AI 单方面改 `.mcp.json`/配置就用。
4. **库 / CLI 可直接用**，但遵守安全底线：装依赖属"需批准"，先说明。

## B. 把可复用能力沉淀成新技能
一段**验证过、以后还会用**的流程（不是一次性操作），沉淀成新技能：
1. 新建 `.agents/skills/<名>/SKILL.md`，frontmatter 写清 `name` + `description`——**description 必须讲清「何时该用」**（触发词要准，这是它被自动调用的唯一依据）。
2. 正文写"怎么干"：简洁、给可直接复制的命令，链接细节而非堆砌。**对齐官方 SKILL.md 规范**——正文保持精简（<~500 行）；重内容拆到技能目录下的 `scripts/`（可执行脚本）、`references/`（详细文档）、`assets/`（模板），由正文按需引用，进一步省上下文。
3. 在 `AGENTS.md` 的「技能索引」加一行：场景 → 新技能。
4. **不要塞进 `AGENTS.md` 正文**——它要常驻精简（<150 行），细节一律进 skills（按需加载）。
5. **如实报告**：新增/改了哪个技能、为什么。

## C. 内容变多时：整理，但安全地清
触发：技能/规则明显冗余、重复、过时，或 `AGENTS.md` 超长被淹没。
1. **判定"垃圾"要有依据**（不靠猜）：与现有重复、所引用的文件/工具已不存在、明显被取代、长期无场景可用。
2. **按破坏性分级处理**（遵守安全底线——删除是破坏性操作）：
   - 默认**先列出候选 + 逐条说明理由 → 交用户确认**再动。
   - 可逆优先：移到 `.agents/skills/_archive/` 而非硬删。
   - **绝不无确认地批量删除。**
3. 同时合并重复、给 description 去歧义、把 `AGENTS.md` 超长部分下沉到技能。
   - 整理时先跑 `node .agents/hooks/verify-kit.mjs` 查一致性（索引↔技能、配置↔脚本）。
   - 读 `.agents/.audit.jsonl` 看 hook 命中情况：误伤率低的规则可建议从 warn 升到 deny，常被绕过/无效的可建议下调或删。
4. **范围只限 ai-kit 自身**的规则/技能/文档文件——**绝不碰项目代码、不影响当前正在做的功能与需求**；与当前任务相关或正在用的一律不动，拿不准就保留。
5. **如实报告**整理了什么、删/归档了什么及理由。
6. 清理完成后，更新提醒计时：写入 `.agents/.last-maintenance-reminder` 为当前时间戳（让下次提醒从现在重新计 N 天）。

### 提醒节奏
- `maintenance-reminder` 这个 SessionStart hook 会**隔 N 天（默认 14，`AI_KIT_MAINT_DAYS` 可改）在会话开始时提醒你一次**是否整理。
- 它**只提醒、不删任何东西**；你说"按 self-evolve 整理"才执行上面 C 的流程，且仍要逐条确认。

## D. 沉淀项目约定（轻量 standards）
- 项目里反复用到、却散落的隐性约定（命名、目录布局、错误处理范式、技术栈版本、"好范例"文件位置），随手沉淀进 `AGENTS.md` 末尾「项目特定约定」段，或一份 `tmp_docs/_standards.md`。
- **触发**：发现自己/用户在重复解释同一约定时；或被纠正后（把教训固化下来）。
- **与"沉淀技能"区别**：技能 = 可复用的"怎么干流程"；standards = 本项目特有的"是什么/什么约定"。
- **保持轻量**：一份可读清单 + 指向好范例文件即可，不是数据库；过时就清。和工作区记忆互补（记忆跨会话，standards 随项目走）。

## 关于"自动"（诚实边界）
- AI 只在与你对话的回合里按本技能行动，**没有后台自动循环**。
- 要真正"定时自动检索/沉淀/清理"，需另外设置**定时 agent**（Codex routines / Claude 计划任务）或 **hooks** 来周期触发本技能——这层由用户决定是否开启。
